            # DORA Metrics

## Overview

NovaPay measures software delivery performance using the four DORA metrics.

---

## Deployment Frequency

Definition:
Number of successful deployments to production.

Target:
Multiple deployments per day.

Measurement:
Count of production deployments per day.

---

## Lead Time for Changes

Definition:
Time from code commit to production deployment.

Target:
Less than 1 hour.

Measurement:
Commit timestamp to deployment timestamp.

---

## Change Failure Rate

Definition:
Percentage of deployments causing incidents or rollbacks.

Target:
Less than 5%.

Measurement:
Failed deployments divided by total deployments.

---

## Mean Time To Recovery (MTTR)

Definition:
Time required to recover from a production failure.

Target:
Less than 1 hour.

Measurement:
Incident start time to service restoration time.

---

## Additional Metrics

* Build Success Rate
* Security Gate Pass Rate
* Rollback Frequency
* Deployment Duration
* Compliance Gate Success Rate

## Expected Outcome

The NovaPay DevSecOps pipeline is designed to achieve elite DORA performance levels through automation, observability, and controlled deployment strategies.
